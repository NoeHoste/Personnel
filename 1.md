## 📋 Matrice de Dépendance

### Pour LOGIN (Synchrone)

```
LoginViewController
    ↓ handleLogin()
LoginController
    ↓ attemptLogin()
LoginService
    ↓ login(username, password)
ClientNetworkService
    ↓ sendRequest("LOGIN:...")
    ↑ retourne String response
LoginService
    ↓ parse réponse
    ↓ crée Teacher ou Player
LoginViewController
    ↓ navigue selon rôle
```

### Pour CREATE_QUIZ

```
CreateQuizViewController
    ↓ saveQuiz()
CreateQuizController
    ↓ saveEntireQuiz()
QuizService
    ↓ saveQuizComplete(title, questions)
    ├─ SessionContext.getCurrentUser()
    ├─ Crée JSON avec Gson
    └─ ClientNetworkService.sendRequest("CREATE_QUIZ:...")
        ↓ TCP vers serveur
ClientHandler (serveur)
    ├─ Parse JSON
    ├─ UserDAO.getIdByUsername()
    ├─ QuizDAO.createQuiz() ────→ INSERT INTO quiz
    └─ Pour chaque question:
        ├─ QuestionDAO.insert*Question() ──→ INSERT INTO question
        └─ QuizDAO.addQuestionToQuiz() ────→ INSERT INTO quiz_question
```

## 🔀 Flux de Données vs Flux de Contrôle

### Flux FORWARD (données vers serveur)

```
Utilisateur input
    ↓
JavaFX event handler
    ↓
ViewController.handleEvent()
    ↓
Controller.method()
    ↓
Service.method()
    ↓
Network (CLIENT)
    ↓ TCP MESSAGE
    ↓
Network (SERVER)
    ↓
ClientHandler.run()
    ↓
Server Service/DAO
    ↓
Database
```

### Flux BACKWARD (réponse vers client)

```
Database
    ↓
DAO.method() retourne objet
    ↓
Server Service
    ↓
ClientHandler.out.println(response)
    ↓ TCP RESPONSE
    ↓
Network (CLIENT)
    ↓
ClientNetworkService retourne String
    ↓
Client Service
    ↓
Client Controller
    ↓
ViewController
    ↓
JavaFX UI update
```

### Flux PUSH (serveur vers client temps réel)

```
GameNetworkService.sendToTeacher(message)
    ↓
ClientHandler (du professeur).out.println(message)
    ↓ TCP PUSH
    ↓
ServerConnection.startListening() (thread d'écoute)
    ↓
IMessageReceiver.onMessageReceived()
    ↓
GameService ou autre implémentation
    ↓
ViewController update UI
```

## 📦 Import Patterns

### Pattern 1 : Service injecte Repository

```java
// Dans ClientHandler
UserService userService = new UserService(new UserDAO());
// Injecte le DAO dans le service
```

**Fichiers impliqués :**
- `UserService.java` → accepte IUserRepository
- `UserDAO.java` → implémente IUserRepository
- `IUserRepository.java` → interface

### Pattern 2 : ViewController utilise Controller

```java
// Dans LoginViewController
private final LoginController coreController = new LoginController();

// Utilise :
IUser user = coreController.attemptLogin(user, pass);
```

**Fichiers impliqués :**
- `LoginViewController.java` → crée instance
- `LoginController.java` → classe métier

### Pattern 3 : Singleton pour ressource unique

```java
// Dans UserDAO et QuestionDAO
PreparedStatement ps = DBConnection.getInstance().prepareStatement(sql);
```

**Fichiers impliqués :**
- `DBConnection.java` → instance unique
- Tous les DAO l'utilisent

## 🌐 Protocoles de Communication

### Format général
```
TYPE:PARAM1:PARAM2:...
```

### Requêtes CLIENT → SERVER (Synchrones)

| Requête | Handler | Service | DAO | Response |
|---------|---------|---------|-----|----------|
| `LOGIN:user:pass` | ClientHandler | UserService | UserDAO | `LOGIN_SUCCESS:role` |
| `REGISTER:user:pass:role` | ClientHandler | UserService | UserDAO | `REGISTER_SUCCESS` |
| `ADD_QUESTION:TYPE:text:answer` | ClientHandler | QuestionService | QuestionDAO | `ADD_SUCCESS` |
| `CREATE_QUIZ:{json}` | ClientHandler | QuizManager | QuizDAO + QuestionDAO | `CREATE_SUCCESS` |

### Messages CLIENT → SERVER (Asynchrones - Jeu)

| Message | Handler |
|---------|---------|
| `JOIN:pin:pseudo` | ClientHandler → stocke clientPseudo |
| `ANSWER:questionId:answer` | ClientHandler → StudentAnswer DAO |

### Messages SERVER → CLIENT (Push)

| Message | Source | Consommateur |
|---------|--------|--------------|
| `QUIZ_START:quizId` | GameNetworkService | GameService → affiche questions |
| `QUESTION:id:text:type` | GameNetworkService | GameService → affiche question |
| `PLAYER_JOINED:count:pseudo` | ClientHandler | DashboardViewController → met à jour compte |
| `GAME_ENDED:results` | GameNetworkService | GameService → affiche résultats |

## 💾 Tables Base de Données

### Relation avec les classes Java

```
DB Table: user
├─ id, username, password, role
└─ Classe Java: AbstractUser, Teacher, Player

DB Table: question
├─ id, type, text, correct_answer, options
└─ Classes Java: Question, TrueFalseQuestion, MultipleChoiceQuestion, OpenQuestion

DB Table: quiz
├─ id, title, teacher_id
└─ Classe Java: Quiz

DB Table: quiz_question (relation)
├─ quiz_id, question_id, question_order
└─ Utilisé par: QuizDAO.addQuestionToQuiz()

DB Table: quiz_session
├─ id, code, user_id, quiz_id, is_completed
└─ Classe Java: QuizSession (implicite)

DB Table: student_answer
├─ id, quiz_session_id, question_id, student_name, answer, is_correct
└─ Classe Java: StudentAnswer
```

## 🔍 Tracing d'une Requête Complète

### LOGIN Flow

```
1. LoginViewController.handleLogin() [UI]
   ↓
2. LoginController.attemptLogin() [Contrôle]
   ↓
3. LoginService.login() [Service]
   ↓
4. ClientNetworkService.sendRequest("LOGIN:...") [Network CLIENT]
   ↓ TCP SEND
5. ═══════════════════════════ INTERNET ═══════════════════════════
   ↓ TCP RECEIVE
6. ClientHandler.run() [Network SERVER]
   ↓
7. UserService.login() [Service SERVER]
   ↓
8. UserDAO.findByCredentials() [DAO]
   ↓
9. DBConnection.getInstance().prepareStatement() [DB]
   ↓
10. MariaDB: SELECT id, username, role FROM user WHERE username=? AND password=?
    ↓ Résultat: id=5, username=prof123, role=teacher
11. UserDAO retourne Teacher(prof123)
    ↓
12. UserService retourne Teacher
    ↓
13. ClientHandler envoie "LOGIN_SUCCESS:Teacher"
    ↓
14. ClientNetworkService reçoit réponse
    ↓
15. LoginService parse et crée Teacher
    ↓
16. LoginViewController.handleLogin() reçoit Teacher
    ↓
17. SceneManager.switchScene("/fxml/dashboard.fxml")
    ↓
18. Dashboard affichée
```
